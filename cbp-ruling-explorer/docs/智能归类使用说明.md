# 智能 HTSUS 归类使用说明

## 1. 配置模型

编辑 `backend/config.json` 的 `models`：

```json
{
  "chat": {
    "base_url": "https://聊天模型服务/v1/chat/completions",
    "api_key": "聊天模型 API Key",
    "model": "支持结构化 JSON 输出的聊天模型",
    "timeout_seconds": 120
  },
  "embedding": {
    "base_url": "https://嵌入模型服务/v1/embeddings",
    "api_key": "嵌入模型 API Key",
    "model": "Embedding 模型名称",
    "timeout_seconds": 120,
    "batch_size": 64
  },
  "reranker": {
    "base_url": "完整的重排序接口 URL",
    "api_key": "重排序模型 API Key",
    "model": "专用 Reranker 模型名称",
    "timeout_seconds": 120
  }
}
```

该文件已加入 `.gitignore`，不会提交密钥。`config.example.json` 是可提交的配置模板。聊天、嵌入和重排序模型均直接读取各自配置块中的完整 URL、API Key 和模型名。

## 2. 初始化索引

在 `cbp-ruling-explorer/backend` 下执行：

```powershell
python -m app.rag sync
python -m app.rag status
```

`sync` 会下载当前配置的 USITC HTS JSON，并按正文哈希增量切分、嵌入 CBP 裁定。再次运行时，未变化的裁定不会重复生成向量。爬虫任务成功结束后也会自动执行裁定索引增量同步；模型配置不完整时会跳过，并在爬虫状态中提示。

RAG 数据默认写入 `cbp-ruling-explorer/data/rag/rag_index.db`，与爬虫生成的 `cbp_rulings.db` 分离。

## 3. 启动和使用

按项目原方式启动 FastAPI 与 React，访问 `/classify`。至少填写产品名称和完整描述；材料、部件、功能、用途和技术规格越完整，判断越可靠。

相关接口：

- `GET /api/classify/index-status`：查看案例切块、裁定数、HTS 条目和版本。
- `POST /api/classify`：检索案例、重排、校验现行 HTS 并生成归类建议。

模型不能自由生成案例编号。后端只接受本次召回的裁定编号，并根据切块 ID 回填原文；主推荐税号必须存在于当前 HTS 索引。

输出仅用于案例辅助和初步判断，不构成 CBP 正式约束性裁定。
