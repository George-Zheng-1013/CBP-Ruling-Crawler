## 关键发现 — 现有爬虫抓不到线上数据，需改走 JSON API
- 用户打开前端后问"如何爬取"。澄清：前端(cbp-ruling-explorer)是只读查询，不爬取；爬取归 cbp-crawler。
- 实测 rulings.cbp.gov：`/search` 是 Angular SPA 空壳(仅8.9KB、含app-root)，requests+BeautifulSoup 解析拿不到任何裁定 → 现有爬虫 `python main.py --phase all` 直接跑会抓到 **0 条**（fetcher/extractor 按静态HTML写的，是当初"阶段一页面结构调研"未对线上验证的后果）。
- **已定位真实官方 JSON API（无需 Playwright）**：
  - 搜索: `GET https://rulings.cbp.gov/api/search?term=&pageSize=&page=&sortBy=RELEVANCE` → `{"rulings":[...],"totalHits":N}`；每条含 `rulingNumber/subject/rulingDate/collection/tariffs(HSCODE数组)/categories/relatedRulings/...`
  - 详情: `GET https://rulings.cbp.gov/api/ruling/{rulingNumber}` → JSON(~2.5KB，含全文)
  - term=fitness 实测 totalHits=334，首条 H88386 tariffs=['9506.91.0030']。
- 结论：需对 cbp-crawler 做增量改造(fetcher 打 API + extractor/parser 改解析 JSON)，字段几乎直取、比 HTML 解析更稳。待用户拍板是否启动团队改造。
