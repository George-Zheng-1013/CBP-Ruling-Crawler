## 裁定号前缀澄清（用户疑问：为何全是 H/N 开头）

用户质疑库里都是 H/N 开头而非 HQ/NY。经 API 实测确认：**数据完全正确**。
- CBP CROSS 系统的裁定编号用**单字母前缀**：`H` = HQ 系列（总部裁定，collection=hq），`N` = NY 系列（纽约裁定，collection=ny）
- "HQ"/"NY" 是**系列名称/签发办公室**（HQ总部、NY纽约），不是裁定号前缀
- API 实测：H328693→collection=hq；N316829→collection=ny
- 任务书自身示例就是 `N082097`（N 前缀，非 NY），印证此格式
- 库内分布：N 前缀 496 条（NY 系列）、H 前缀 88 条（HQ 系列）、HQ*/NY* 前缀各 0 条
- 结论：这正是任务书要求的"关税归类裁定 HQ/NY/N 系列"数据，无需修改
- html_store/task_queue 是旧 HTML 管线的支撑设施，JSON 路径不用，一并移除以符合"不留旧 HTML 逻辑"
- utils.py 的 normalize_url/extract_ruling_no_from_url 为纯字符串工具，保留（test_utils 仍覆盖）

在前端 AppHeader 增加两个操作按钮，后台通过 FastAPI 子进程触发爬虫。

### 后端改动
- `config.py`：新增 CRAWLER_PYTHON / CRAWLER_MAIN_DIR / CRAWLER_LOG_FILE / CRAWL_TIMEOUT_SECONDS
- `.env`：新增 CRAWLER_PYTHON / CRAWLER_MAIN_DIR
- `schemas.py`：新增 CrawlSyncRequest / CrawlJobStatus / CrawlStartResponse 模型
- `routers/crawl.py`（新建）：3 个端点
  - POST /api/crawl/incremental — 检测 MAX(ruling_date)，启动爬虫子进程
  - POST /api/crawl/sync — 用户指定 min_date，启动爬虫子进程
  - GET /api/crawl/status — 返回当前任务状态 + 日志尾部
  - 模块级单例锁，同一时间只允许一个爬虫任务运行；threading + subprocess.Popen 后台执行
